NBA Synergy Engine (Moneyball 2.0)

Permutation-Invariant Neural Network for Lineup Chemistry

Python ¬∑ PyTorch ¬∑ SQL ¬∑ Streamlit Post-graduate personal project

Traditional NBA analytics evaluate players in isolation. This project treats basketball as a chemistry problem: the value of a player depends on who they share the floor with.

What it does:

  • Classifies playstyles using Gaussian Mixture Models (GMM) on player tracking data.
  • Uses a permutation-invariant DeepSet architecture to predict the Net Rating of any 5-man unit based on the vector sum of their playstyles.
  • Implements a ‚ÄúGenerative GM‚Äù module that searches the league for the mathematically optimal 5th starter for a given 4-man core.

Technical architecture:

  • Data Engineering: Ingested 170,000+ possessions over 11 seasons (2014‚Äì2025) from NBA APIs; migrated raw CSVs into a normalized SQLite database (Players, Lineups, Archetypes) for O(1) query access during inference.
  • Unsupervised Learning: Applied PCA + GMM to 12 tracking metrics (speed, micro-touches, dribbles per touch, etc.) to learn 10 latent archetypes (e.g., heliocentric creators, rim runners, traditional facilitators).
  • Deep Learning: Built a custom DeepSet Neural Network in PyTorch to ensure permutation invariance ‚Äî the model treats {Luka, Kyrie} the same as {Kyrie, Luka}.
  • Performance: Trained on 3,500+ qualified lineups to predict Net Rating, achieving an RMSE of ~39.9 on held-out data.

Optimization module (‚ÄúGenerative GM‚Äù):

  • Takes a 4-man core (e.g., Shai Gilgeous-Alexander, Jalen Williams, Chet Holmgren, Lu Dort).
  • Scans 450+ NBA players and simulates all possible 5th-player combinations via batch inference.
  • Returns the player archetype (and example players) that maximize predicted Net Rating.
  • Validated by showing, for example, that a rim-running big (Cluster 2) best complements OKC‚Äôs ball-dominant core.
